Q. Less obvious perks of using 'live' data resources include logins and payments. How could Twitter be used in these two scenarios?

A couple of options come to my mind when using a login for Twitter, the biggest one is it gives a way for your app to be able to verify and authenticate who the user is. Secondly it can be used to link your app with the persons Twitter account, much like many games now allow you to login using Facebook instead of creating an account with the game or just playing as a guest. In terms of sending payments using Twitter I actually found a company using a Google search that has done just this, it is called Dwolla and all it requires is a person to setup an account at the Dwolla website to link their Twitter account and then all a person needs to do is tweet someone with a dollar amount and hash-tag Dwolla. I could see a lot of apps using something like this, especially one designed for selling good to people, no longer would someone need multiple accounts, just have a Twitter account to integrate with the app and easily buy and sell goods in your own app.

Q. Visit Programmableweb.com and search for JSON-P data sources. Why would we need JSON-P, specifically?

We need to use JSON-P as developers because it allows us to access data from outside domains which is typically restricted by web browsers because of the same origin policy we learned about in a previous course. This in turn empowers us to pull that data from the other domain and display it in our own website/app, one of the main uses of an API.

Q. Visit a video service on the web (like YouTube) and search for "Screen Reader Demo" and "Mobile Screen Reader" - what are some of the common frustrations (or common support mechanisms) that you see among the examples that are given - how could you fix them (or use them)?

The improper use or not using headers at all on your website, headers give importance to text on the screen, for example the titles of articles on a news website. If you put random information in headings then the user will think that the information being read to them has some importance to them. There is also the example of not using headings at all, in which case a user may not even know where to begin to find any information, or worse yet may even be lead to believe that there is no information on the website at all. Another common frustration is not using alt tags on images, these should always be used to give a brief description of the image failure to do so will leave users with no information about what the image is, and even for people who do not need a screen image if the image cannot be found at least there will be some kind of description as to what should have been in that location.

Using labels on form elements is also crucial especially if your app or website needs a user to submit a form before they can properly use the website. If no label is given for the related form element then the user will not be told about the form element and what the required input is. Always been told the form failed when not actually being able to fill it out properly can grow extremely frustrating, very quickly. A lot of issues are also raised about links and how descriptive they are, if you have links you need to make them as descriptive as possible as they will let the user know exactly where they are going on your website. Random links thrown around a website with very little in the way of description makes it easy for a user to get lost while navigating and can even miss the area of your website that they intended to come as use so at all times it is extremely important to be as precise and descriptive as possible.
